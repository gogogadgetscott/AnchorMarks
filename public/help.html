üöÄ Getting Started

Welcome to AnchorMarks

AnchorMarks is a powerful, self-hosted bookmark manager designed to help you organize, search, and access your bookmarks efficiently. Whether you're managing dozens or thousands of bookmarks, AnchorMarks provides the tools you need.

Key Features

üìÅ Smart Organization

Organize bookmarks with folders, tags, and smart collections. Create nested folder hierarchies and use color-coding for visual organization.

üîç Powerful Search

üìë Managing Bookmarks

Adding Bookmarks

There are several ways to add bookmarks to AnchorMarks:

  1. Manual Entry: Click the "+" button or press Ctrl+N (Cmd+N on Mac)
  2. Browser Extension: Save bookmarks directly from your browser
  3. Import: Import from HTML exports from other browsers

Editing Bookmarks

Click the edit icon on any bookmark card to modify its details:

  • Title: The display name for your bookmark
  • URL: The web address
  • Description: Optional notes about the bookmark
  • Folder: Organize into a specific folder
  • Tags: Add multiple tags for categorization

Favorites

Mark important bookmarks as favorites by clicking the star icon. Access all favorites quickly from the sidebar navigation.

Bulk Operations

Select multiple bookmarks to perform batch operations:

  • Move to folder
  • Add/remove tags
  • Mark as favorite
  • Delete multiple bookmarks
üí° Tip: Press Ctrl+A (Cmd+A on Mac) to select all visible bookmarks. Use Shift+Click to select a range.

üìÅ Organizing with Folders

Creating Folders

Create folders to organize bookmarks hierarchically:

  1. Click "Add Folder" in the sidebar
  2. Choose a name and color
  3. Optionally select a parent folder for nesting

Folder Features

  • Colors: Assign colors for quick visual identification
  • Nesting: Create unlimited nested folder levels
  • Drag & Drop: Move bookmarks between folders by dragging
  • Quick Jump: Press Ctrl+1 through Ctrl+9 to jump to top-level folders

Folder Navigation

Navigate folders from the sidebar or use keyboard shortcuts. The folder tree shows bookmark counts and can be collapsed/expanded for cleaner organization.

üè∑Ô∏è Using Tags

Adding Tags

Tags provide flexible categorization beyond folders. Add multiple tags to any bookmark:

  • Enter tags separated by commas when creating/editing bookmarks
  • Use autocomplete to select existing tags
  • Create hierarchical tags using slashes: dev/javascript

Tag Features

  • Hierarchical Tags: Use parent/child syntax for nested tags
  • Tag Suggestions: Auto-suggest tags based on bookmark URLs and existing tags
  • Bulk Tagging: Add or remove tags from multiple bookmarks
  • Tag Renaming: Rename/merge tags across all bookmarks from Settings

Tag Management

View tag statistics in Settings ‚Üí Tags to see usage counts and rename/merge tags. This is useful for cleaning up duplicate or similar tags.

üí° Tip: Use hierarchical tags for better organization. For example: tech/programming/javascript creates a clear structure.

ü§ñ Smart Organization

Overview

Smart Organization uses AI-assisted algorithms to automatically suggest tags, create collections, and provide insights about your bookmarks. It learns from your existing bookmarks and usage patterns to make intelligent recommendations.

Smart Tag Suggestions

When adding or editing a bookmark, get intelligent tag suggestions based on three factors:

  • Domain Analysis: Tags commonly used for similar websites (e.g., GitHub ‚Üí dev, code)
  • Activity Patterns: Tags you've used recently when bookmarking similar content
  • Content Similarity: Tags from your existing bookmarks with similar titles or descriptions

Each suggestion includes a confidence score (0-100%) and explanation of why it was suggested.

Smart Collections

Auto-generate bookmark collections based on intelligent analysis:

  • Domain-Based: Collections for specific websites (GitHub, Stack Overflow, etc.)
  • Activity-Based: Collections for recent bookmarks, frequently clicked, or unread items
  • Tag Clusters: Semantic groupings of related tags

Create suggested collections with one click from the Smart Collections widget.

Smart Insights Dashboard

Get a comprehensive analytics view of your bookmark organization:

  • Total bookmarks, tags, and click statistics
  • Top domains visualization and click patterns
  • Most used tags and tag frequency distribution
  • Activity timeline and engagement metrics
  • Smart suggestions for organization improvements

How Smart Organization Works

The system uses a 3-factor scoring algorithm:

Score = (Domain √ó 35%) + (Activity √ó 40%) + (Similarity √ó 25%)
  • Domain Score: How frequently a tag is used for similar domains
  • Activity Score: How recently you've used a tag with similar bookmarks
  • Similarity Score: How closely a tag matches the bookmark's content
üí° Tip: The more bookmarks you have and the more consistently you tag them, the better the suggestions become. Trust the suggestions to save time on organization!

üîå API Access

Getting Your API Key

  1. Go to Settings ‚Üí API Access
  2. Copy your API key
  3. Use it in API requests or third-party integrations

API Endpoints

Search Bookmarks

GET /api/quick-search?q=query&api_key=YOUR_KEY

Returns matching bookmarks optimized for quick search (Flow Launcher).

Get All Bookmarks

GET /api/bookmarks?api_key=YOUR_KEY

Returns all bookmarks for the authenticated user.

Create Bookmark

POST /api/bookmarks?api_key=YOUR_KEY Content-Type: application/json { "url": "https://example.com", "title": "Example", "tags": "tag1,tag2", "folder_id": "folder-uuid" }

Get Folders

GET /api/folders?api_key=YOUR_KEY

Flow Launcher Plugin

AnchorMarks includes a Flow Launcher plugin for quick bookmark access:

  1. Install Flow Launcher
  2. Copy the plugin from flow-launcher-plugin/ directory
  3. Configure with your server URL and API key
  4. Use am query to search bookmarks
üîí Security: Keep your API key secure. It provides full access to your bookmarks. Regenerate it from Settings if compromised.

üöÄ Deployment & Configuration

Production Environment Setup

Configure environment variables for production deployment:

Required Variables

  • NODE_ENV: Set to production
  • JWT_SECRET: Strong random string (app refuses to start without this)
  • DB_PATH: Absolute path to SQLite database (e.g., /data/anchormarks.db)
  • CORS_ORIGIN: Comma-separated allowed origins (‚ö†Ô∏è cannot be *)

Optional Variables

  • PORT: Server port (default: 3000)
  • HOST: Bind address (default: 0.0.0.0)
  • SSL_KEY: Path to SSL private key (only if not using reverse proxy)
  • SSL_CERT: Path to SSL certificate

Docker Deployment

The easiest way to deploy AnchorMarks:

docker-compose up -d

Or manually:

docker build -t anchormarks:latest . docker run -d \ --name anchormarks \ -p 3000:3000 \ -v anchormarks_data:/data \ -e NODE_ENV=production \ -e JWT_SECRET=your-secret-key \ anchormarks:latest

Linux/macOS Direct Installation

  1. Install Node.js 18+
  2. Clone or copy AnchorMarks to /opt/anchormarks
  3. Run npm ci --only=production
  4. Create .env file with production settings
  5. Set up systemd service (copy from deploy/anchormarks.service)
  6. Enable with sudo systemctl enable anchormarks
  7. Start with sudo systemctl start anchormarks

Reverse Proxy (Nginx)

Configure Nginx as a reverse proxy for production. A template is included in deploy/nginx.conf.

üîí Security: Always use HTTPS with a valid SSL certificate. Configure your reverse proxy to handle SSL termination.

üì° Complete API Reference

Authentication

Use your API key in requests:

X-API-Key: lv_your_api_key

Find your API key in Settings ‚Üí API Access.

Core Endpoints

Health Check (No Auth)

GET /api/health

Search Bookmarks

GET /api/quick-search?q=github

Get All Bookmarks

GET /api/bookmarks

Create Bookmark

POST /api/bookmarks Content-Type: application/json { "url": "https://github.com", "title": "GitHub", "description": "Code collaboration", "tags": ["dev", "code"] }

Update Bookmark

PUT /api/bookmarks/bookmark-id { "title": "New Title", "description": "Updated" }

Delete Bookmark

DELETE /api/bookmarks/bookmark-id

Get Folders

GET /api/folders

Response Format

All successful responses return JSON with the bookmark/folder object containing id, title, url, tags, and metadata.

üîê Security Best Practices

Built-In Protections

  • Password Hashing: bcryptjs with salt rounds
  • JWT Tokens: 30-day expiry with httpOnly cookies
  • CSRF Protection: Tokens required for state mutations
  • User Isolation: All queries filtered by user_id
  • SSRF Guard: Private IP blocking on favicon/link checks
  • SQL Injection Prevention: Parameterized queries throughout
  • Input Validation: URLs, names, tags sanitized before storage

Production Security Checklist

  • ‚úì Set NODE_ENV=production
  • ‚úì Generate strong JWT_SECRET
  • ‚úì Use HTTPS with valid SSL certificate
  • ‚úì Set CORS_ORIGIN to your domain(s) - never use *
  • ‚úì Use absolute DB_PATH (e.g., /data/anchormarks.db)
  • ‚úì Enable firewall - only expose ports 80, 443
  • ‚úì Set up automated database backups
  • ‚úì Rotate API keys periodically
  • ‚úì Monitor disk space and database size

Generating JWT_SECRET

node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
üîí Important: Never share your JWT_SECRET or API keys. Treat them like passwords. If exposed, regenerate immediately from Settings.

üîß Troubleshooting

Common Issues

Server won't start

  • Check that Node.js 18+ is installed: node --version
  • Verify dependencies: npm install
  • Check environment variables in .env file
  • Ensure JWT_SECRET is set (production mode)

Can't access the app

  • Verify server is running: curl http://localhost:3000/api/health
  • Check firewall settings
  • For remote access, ensure reverse proxy is configured
  • Verify CORS_ORIGIN includes your domain

Database issues

  • If database is locked, restart the server
  • Check file permissions on database file (should be 600)
  • Ensure DB_PATH is an absolute path on production
  • Set up automated backups regularly

Browser extension not working

  • Verify extension is installed in developer mode
  • Check API key is correct in extension settings
  • Ensure server URL is accessible from your browser
  • Check browser console for error messages

API requests failing

  • Verify API key is correct and not expired
  • Check X-API-Key header is included
  • For CORS errors, verify CORS_ORIGIN configuration
  • Check server logs for detailed error information
