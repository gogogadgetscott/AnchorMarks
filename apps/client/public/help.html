Editing & favorites

  • Edit a bookmark‚Äôs title, URL, description, folder, and tags.

Bulk actions & Selection Mode

Select multiple bookmarks to perform actions on all of them at once. When one or more bookmarks are selected, the header transforms into Selection Mode.

  • Enter selection mode: Click the checkbox in the top-left corner of any bookmark card.
  • Multi-select: Use Ctrl/Cmd+click to add individual items, or Shift+click to select a range.
  • Selection Header: The blue header displays the current count and provides tools for bulk archiving, moving, tagging, or deleting.
  • Clear selection: Click the "X" in the selection header or press Esc.

üì¶Archiving

Archive bookmarks you don't need immediately but want to keep. Archived bookmarks are hidden from main views (All, Folders, Favorites) but remain searchable.

  • Archive: Click the archive icon (box) on any bookmark card or use the bulk action bar.
  • Archived View: Access all your archived items via the "Archived" link in the sidebar.
  • Restore: Click the unarchive icon (box with up arrow) in the Archived view to return a bookmark to its original place.

Bookmark Colors

Assign a custom color to any bookmark for visual organization. Colors appear as a vibrant left border accent and subtle background tint on bookmark cards.

  • Setting a color: When adding or editing a bookmark, click a color swatch from the color picker below the tags field.
  • Color palette: Choose from 16 vibrant colors including blue, indigo, purple, pink, rose, orange, amber, yellow, lime, green, emerald, teal, cyan, sky, and slate. Use the crossed circle to remove color.
  • Visual display: Colored bookmarks show an 8px left border in the selected color, plus a subtle background tint that adapts to light and dark themes.
  • Dashboard widgets: Colors also appear on bookmarks within dashboard widgets, with a 6px left border accent.
üí°Color ideas: Use colors to mark priority levels (red = urgent, yellow = review), status (green = active, gray = archived), or categories (blue = work, purple = personal). Colors work alongside folders and tags for maximum flexibility.

üëÅÔ∏èAt a glance: The Tag Cloud is an interactive visualization that displays all your tags sized by their usage frequency, making it easy to see your most-used tags at a glance and quickly filter bookmarks.

  1. Open Settings ‚Üí Views ‚Üí Import.
  2. Choose a previously exported JSON file (from AnchorMarks).
  3. Review the preview: the importer shows the view name, rules, and (for dashboards) number of widgets and their grid positions. You may optionally rename the imported view.

  • Keep a copy of your exported JSON as a backup of custom dashboards and complex views.

üîêSecurity & Auth

This section consolidates security best practices and the auth/CSRF model used by the web UI and API.

Built-in protections

  • Password hashing (bcrypt)
  • JWT auth in an httpOnly cookie for browser sessions
  • CSRF tokens required for state-changing browser requests
  • User isolation: every query filters by user_id
  • SSRF guard for remote fetches (blocks private IP ranges)

Developer: Auth & CSRF (how it works)

  • JWT cookie: login/register sets a token cookie; browser requests include it automatically.
  • CSRF: mutating requests send X-CSRF-Token matching the csrfToken cookie.
  • API keys: integrations authenticate with X-API-Key and bypass CSRF (non-cookie requests).

Development Workflow (Vite)

AnchorMarks uses Vite for frontend development with Hot Module Replacement (HMR). In development mode, two servers run simultaneously:

  • Express Backend: Runs on port 3000 (configurable via PORT in apps/.env)
  • Vite Dev Server: Runs on port 5173 (configurable via VITE_PORT in apps/.env)

Available Scripts

# Install dependencies npm install # Start both dev servers (recommended for development) npm start # or explicitly: npm run dev:full # Start only the backend npm run dev # Start only Vite dev server npm run dev:vite # Stop both servers npm run stop # Restart both servers npm run restart # Build for production npm run build # Run in production mode npm run prod # Run tests npm test
üí°Development Mode: Access the app at http://localhost:5173 (Vite dev server). All API requests are automatically proxied to the Express backend on port 3000. Changes to frontend code trigger instant HMR updates without full page reloads.
üí°Production Mode: Run npm run build to compile the frontend into apps/client/dist. Then npm run prod serves everything from the Express server on a single port (3000 by default). No Vite dev server needed in production.

Port Configuration

Edit apps/.env to customize ports:

# Backend server port PORT=3000 # Vite dev server port VITE_PORT=5173

The Vite configuration (apps/vite.config.js) automatically reads these values and configures the proxy to route API requests to the correct backend port.

‚ùìFAQ

How do I backup my bookmarks?

Use the export feature in Settings ‚Üí Export to download your bookmarks as JSON. For dashboard layouts and views, use Settings ‚Üí Views ‚Üí Export.

Can I import bookmarks from my browser?

Yes! Use Settings ‚Üí Import ‚Üí Browser Bookmarks. AnchorMarks supports HTML bookmark files from Chrome, Firefox, Edge, and Safari.

How do I organize bookmarks with folders and tags?

Folders are for hierarchical organization (like file folders), while tags are for flexible categorization (one bookmark can have multiple tags). Use both together for maximum organization.

What's the difference between Smart Collections and Views?

Smart Collections are dynamic groupings based on rules (e.g., all GitHub links). Views are saved filter/sort states that you can restore later.

How do I reset my password?

If you're self-hosting, passwords are managed locally. For the hosted version, use the "Forgot Password" link on the login page.

Can I use AnchorMarks offline?

The web app requires a server connection, but you can self-host it on your local network for offline use.

How do I customize the dashboard?

Click "Add Widget" to add folder or tag widgets. Drag to rearrange, resize handles to adjust size, and use the widget options menu (‚ãÆ) for per-widget settings.

Why aren't my favicons loading?

Favicons are fetched automatically when adding bookmarks. If they're missing, the server may be blocking external requests or the site doesn't provide favicons.
